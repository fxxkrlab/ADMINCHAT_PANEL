# ADMINCHAT Panel - 原始开发需求

> 本文档为项目原始需求记录，不可修改，仅供对照参考。

## 项目目的

开发一款 Telegram 双向信息转发 BOT（包括信息、媒体等，例如：图片、文字双向转发后仍旧要保持正确的 Telegram 指定可用的 Markdown 格式内容）

## 背景

市面上没有特别好用的。

## 详细需求

### 1. Web 端消息管理
我需要这个 Bot 在被对方私聊或者在群里 @ 后收到的信息，能够在 WEB 分群组（群组名）（群组内用户名）/ 单独用户进行管理。

### 2. 用户多维度管理
多维度可设置，例如：block 用户（拉黑他与 telegram bot 的关系），unblock 用户，在 web 页面给这个用户加 tag 标签显示在名字后面，分组用户等。

### 3. Web 聊天框回复
并且可通过 web 页面聊天框进行回复。（聊天框内要能够正常收到 TG 的信息、媒体图片等，也要能够让我正常能够回复发送信息、图片并且能够用 telegram 允许的 markdown 等）

### 4. 用户搜索
可以对用户进行 TGUID 搜索然后展示这个用户 telegram 的所有 info 信息，可以对上面提到的给用户加的 tag 搜索。

### 5. 多 Bot 池与限流处理
这个 web 可以控制多个 telegram bot，私聊时每个 bot 都可以双向转发消息（哪个 bot 收到的来自用户的消息，就从哪个 bot 转发回去），但是另一种情况是在群里，telegram bot 发消息会受到频率限制，为了应对此情况，比如 bot 接收消息（被用户在群里 @），web 端管理员回复内容后，也可以通过 bot1 reply 提问题的用户，但是内部要检测如果 bot 被 telegram 限流了，则通过 bot2 回复，或者 bot3，要确保这个消息只被 1 个 bot 回复过！bot 池可以被管理员无限添加。但是群内接收消息必须 @ 这个 bot 才行，谁被 @ 优先从原路回复并且 reply 用户，只有当 bot 限流后会随机到 bot 池内别的 bot。

对于 webpanel 里管理员看到的是统一的聊天框，每个用户的对话框都单独生成一个侧边栏，可以上下滚动，有新消息的跳到上面来：
- 如果是群里用户 @ bot 发送的消息则前面多一个什么群的提示，例如：`群HALO - 用户xxx`（下排显示 tguid）
- 如果单纯是用户私聊，则直接显示（上排用户名）（下排 tguid）

### 6. 管理员与客服系统
这个 webpanel 是有用户设置功能的，admin 可以添加管理员允许登录这个 panel，然后帮助回复（相当于一个 web 客服系统）。

### 7. 统计栏
系统上面需要有一个统计栏：
- 已接收了多少私聊
- 处理了多少（聊天框可以手动标记为已处理，如果用户又发新的内容了则跳回未处理）
- 未处理的有多少
- 黑名单了多少（也需要黑名单列表）

### 8. 用户信息 Grid 页面
每个私聊的都是以侧边栏的方式在聊天框页面存在，但是也要提供一个 grid 页面，显示私聊过或者 @ 过 bot 的用户头像、telegram 的所有信息（例如 tguid、其他 info、Datacenter、IP 地区等一切能够获得的信息）。

### 9. 权限系统
可能会有考虑不到的权限方面的问题，可以简单设置一些客服系统应当有的权限。

### 10. 数据库选型
这个系统的数据库应该用什么保存？PostgreSQL 还是 MongoDB 之类的？

### 11. FAQ 自动回复系统
系统要能够允许设置 FAQ，正则匹配（可选的多种模式）→ 用户提问 → 自动回复：
- 多关键字 → 1 个回答
- 多关键词 → 在范围内随机多种回答
- 1 个关键字 → 多种回答
- 1 个关键字 → 1v1 单独一个答案
- 关键字匹配模式：从头匹配、search、全匹配等多种设置

### 12. FAQ 管理页面
需要有一个添加 FAQ 的页面，分别添加问题和答案，并且多选复选关联（分屏左右两边，左侧蓝色复选框是问题，右侧绿色复选框是答案，来实现 multi 匹配 multi 的功能）。也需要一个已设置好的 FAQ 列表页面，点击编辑后回到左右分屏的 FAQ 添加编辑页面。

### 13. AI 接口集成
需要预留能够对接 AI API（例如 base_url、apikey 等）的接口，在问题回复的方式里加入：
- 通过 AI 直接回复（需要输出字数、感情、温度等设置，防止用户白嫖 AI 聊天）
- 正则匹配问题后获取答案 → 再让 AI 润色一遍的模式
- 等多种模式

### 14. RAG 预留
预留支持 RAG 数据库 → AI 回答的模式，虽然现阶段不使用，但是以后一定会使用。

### 15. FAQ 问题排行榜
增加一个页面，用户问题排行榜（通过 FAQ 内有的问题），成功 FAQ 回答用户了（匹配到关键词了）的现存问题排行榜。

### 16. 遗漏知识点排行榜
增加一个页面，遗漏的知识点排行榜（未通过 FAQ 回答用户但用户问的很多的问题），可以保存问题提取一些相同关键字，每天凌晨 3 点更新一下这个榜单。管理员可以针对这个榜单设置新的 FAQ，设置完后可以把这个遗漏知识点关键词删除。

### 17. 群内 FAQ 排行指令
只有管理员（这个面板有账号的用户），可以在群里发送指令例如 `/FAQRanking`，来显示用户问的最多的 10 个关键词。

### 18. 异步架构
因为肯定会有很多用户私聊或者同时私聊、群内 @，所以肯定要做成异步代码。

### 19. 技术栈选择
用什么做比较好？Rust 还是 Python，还是 TS，还是 C#，还是多语言结合分工？

### 20. Cloudflare Turnstile 验证
如果用户在群里 @ bot 账号池内的 bot 则无所谓，如果是私聊某个账号池内的 bot，希望能够通过 CF 的 Turnstile 进行验证。（时效是多少天？可以设置还是 CF 统一？）

### 21. 项目名称
这个项目就叫做 ADMINCHAT Panel，在 webpanel 后台左上角要显示。

### 22. Docker 部署
需要通过 GHCR 发布镜像，希望在服务器上 Docker 部署。
